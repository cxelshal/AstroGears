Arduino Code Overview

The Arduino code for our gauntlet prototype is responsible for reading data from various sensors, controlling components like the GME 12864-13 OLED and NeoPixel Ring 12, and providing audio feedback using the buzzer. It follows a modular structure with separate functions for each sensor and component.

Gas Sensor Integration

The Gas Sensor (MQ-135) is integrated into the Arduino code to continuously read analog data from the sensor's analog output pin. The code converts the analog readings into meaningful gas concentration values (PPM) and displays them on the GME 12864-13 OLED.

Temperature and Humidity Sensor Integration

The Temperature and Humidity Sensor (Adafruit AM2320) is connected to the Arduino board using the I2C communication protocol. The code communicates with the sensor to obtain temperature and humidity readings, which are then displayed on the GME 12864-13 OLED.

Ultrasonic Distance Sensor Integration

The Ultrasonic Distance Sensor (HC-SR04) is used to detect obstacles and measure distances. The code triggers ultrasonic pulses and calculates the time taken for the sound waves to bounce back, determining the distance to nearby objects. The buzzer makes audio feedback based on the distance readings.

GME 12864-13 OLED Control

The GME 12864-13 OLED is used to display real-time sensor data, including gas concentration, temperature, and humidity. The code updates the OLED display with new data at regular intervals.

NeoPixel Ring 12 Interaction

The NeoPixel Ring 12 visually responds to gas levels, temperature and humidity readings. The code maps the sensor values to different colors and patterns on the NeoPixel Ring, providing intuitive feedback to the rescuer.

Buzzer Audio Feedback

The code controls the buzzer to provide audio alerts when the Ultrasonic Distance Sensor detects obstacles at short distances. The buzzer emits different sounds to warn the rescuer of potential hazards.
