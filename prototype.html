Prototype Details

Welcome to our gauntlet prototype page. Here, we'll provide all the information about our design, sensors, code, and debugging process for the gauntlet.

Main Design Goals

Our team had several design goals for the gauntlet:

  • Enhanced safety features to navigate through the earthquake-stricken building.
  • Real-time monitoring of gas leaks, temperature, humidity, and distance to ensure the safety of the rescuer.
  • Aesthetically pleasing design to boost the credibility of the rescuer.

Sensor Selection

We chose the following sensors for our mission:

  • Gas Sensor (MQ-135): To detect dangerous gas leaks and provide early warnings.
  • Temperature and Humidity Sensor (Adafruit AM2320): To monitor environmental conditions.
  • Ultrasonic Distance Sensor (HC-SR04): To help navigate through dark rooms and obstacles.

Code Design

Our Arduino code was carefully designed to achieve the following:

  • Read data from the sensors: temperature, humidity, gas level, and distance.
  • Display real-time sensor data on the OLED screen.
  • Control the buzzer to provide audio alerts based on distance readings.
  • Adjust the RGB LED color based on the temperature reading.

The code follows a modular approach, with separate functions for sensor readings and distance measurement.

Problem Solving and Debugging

During the development process, we encountered various challenges:

  • Incorrect sensor readings: We used the engineering mindset to carefully check sensor connections and ensure proper wiring.
  • Unexpected behavior: We utilized the Serial Monitor to print sensor readings and troubleshoot issues with the code logic.
  • Iterative testing: We performed multiple iterations to improve the code and hardware based on real-world scenarios.
